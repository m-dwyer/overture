# Core Snapshots and View Models

Overture core will expose snapshots while view modules derive semantic view models and render modules map those view models to OLED and LED presentation. The Project/Scene/Clip refactor should stop adding final screen or LED construction to core so musical decisions, control-surface context, UI derivation, and presentation mapping can be tested separately. Core owns project data, transport state, explicit Control Surface Context, and commands. Overture Project data does not own transient selection or surface mode. View owns selected-cell, grid, route-warning, and playback-status projections from core snapshots; render owns coordinates, raw LED colors, and display drawing.
