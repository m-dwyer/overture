# Retain Inactive Route Data

Overture retains inactive route-specific data instead of deleting it when a Track changes route or a route destination changes shape. Move Routes own a Move Track Target, Schwung Routes own a Schwung Chain, and route-specific Motion is stored with the Overture Clip that owns its timeline; when a route or target is inactive or unavailable, that data is retained for restoration, remapping, or explicit deletion. Core owns user-meaningful route data such as Component roles, module identity, preset identity, parameter values, and unmapped state, while host code owns concrete Schwung slot numbers, host functions, and Move MIDI translation. This favors user trust and reversible exploration over a simpler model that discards destination-specific sound and Motion state during rerouting.
